One Piece Game

Stack: C#, UWP, Game Logic, File Handling

Repository Case Study (PDF)

Overview

A mini adventure game where the player controls Luffy, collects characters (e.g., Zoro, Nami), and avoids obstacles. Features three difficulty levels, a scoring system, and persistent highscores.

Game Features

  • Character movement via keyboard or gamepad.
  • Collision detection and collectible items.
  • Dynamic difficulty (speed and obstacle density).
  • Highscores persisted to local files (JSON or plain text).

Technical Details

  • Canvas rendering with tick-based updates.
  • Game state machine (Start, Playing, Paused, GameOver).
  • File handling for score persistence.
// Simplified game loop
void UpdateGame(TimeSpan delta) {
  player.Move(input, delta);
  foreach (var obj in collectibles) {
    if (player.Collides(obj)) score += obj.Value;
  }
}
